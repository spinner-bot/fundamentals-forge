1. 🗺️ 核心概念速览

1.1 HTML 是什么

HTML（HyperText Markup Language）是网页的骨架。它用标签来标记内容的结构和语义。
CSS 负责"穿什么衣服"（样式），JavaScript 负责"做什么动作"（交互）。
当前标准是 HTML5（2014 年正式发布，持续演进中）。

1.2 DOM 树模型

graph TD DOC["📄 Document
文档根节点"] --> HTML["<html>
根元素"] HTML --> HEAD["<head>
元信息"] HTML --> BODY["<body>
可见内容"] HEAD --> META["<meta>"] HEAD --> TITLE["<title>"] BODY --> H1["<h1>"] BODY --> P["<p>"] BODY --> DIV["<div>"] P --> STRONG["<strong>"] P --> TEXT["文本节点"]

1.3 标签 vs 元素 vs 属性

<a href="https://example.com" target="_blank">点击这里</a>
││                                              │             │
│└────────── 属性 ──────────────────────────────┘             │
│                                                             │
└── 开始标签 ─────────────────────────── 结束标签 ─────────────┘
└──────────────────── 元素 Element ───────────────────────────┘
概念定义
标签 Tag<tagname> 格式的标记符
元素 Element开始标签 + 内容 + 结束标签的完整单元
属性 Attribute标签内的 name="value" 键值对，提供额外信息
空元素 Void Element无结束标签的单标签：<img>、<br>、<hr>、<input> 等

1.4 块级元素 vs 行内元素

特性块级 Block行内 Inline
默认宽度占满父容器宽度由内容决定
换行前后自动换行不换行，跟文字流走

宽高模型width/height 生效width/height 不生效
典型标签<div> <p> <h1> <section><span> <a> <strong> <code>

注：CSS 的 display 属性可以改变元素的显示类型（如 display: inline-block 兼具两者特点）。

2. 📐 文档基础结构

🎯 最小 HTML5 文档 基础 
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>页面标题</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <!-- 页面内容 -->
  <script src="app.js"></script>
</body>
</html>
🏷️ DOCTYPE / html / head / body 基础

💡 一句话：每个 HTML 页面都需要的四个基础结构元素。

标签必要?作用
<!DOCTYPE html>✅ 必须告诉浏览器"这是 HTML5 文档"，触发标准模式渲染
<html>✅ 必须文档根元素，lang 属性指定语言
<head>✅ 必须元信息容器：编码、标题、CSS 引用、SEO 标签
<body>✅ 必须页面上实际显示的所有内容
🏷️ meta 标签 基础

💡 一句话：<meta> 存放浏览器、搜索引擎、社交平台所需的元信息。

🎬 场景⌨️ 代码📝 说明
字符编码（必须）<meta charset="UTF-8">放在 head 最前面，避免乱码
视口（移动端必须）<meta name="viewport" content="width=device-width, initial-scale=1.0">让页面在手机上正确缩放
SEO 描述<meta name="description" content="页面描述(150字内)">搜索引擎结果中的摘要文字
SEO 关键词<meta name="keywords" content="HTML, 教程">（现代搜索引擎已不太看重）
作者<meta name="author" content="作者名">标注页面作者
禁止缓存<meta http-equiv="Cache-Control" content="no-cache">告诉浏览器不要缓存
IE 兼容<meta http-equiv="X-UA-Compatible" content="IE=edge">让 IE 使用最新渲染引擎
Open Graph<meta property="og:title" content="标题">社交分享卡片（微信/FB/Twitter）
🏷️ title / link / base 基础
标签用途示例
<title>浏览器标签页标题（必须，SEO 关键）<title>我的网站 - 首页</title>
<link>引入外部资源（CSS、字体、图标）<link rel="stylesheet" href="style.css">
<link rel="icon">网站图标（Favicon）<link rel="icon" href="favicon.ico">
<base>设置页面所有相对 URL 的基准地址<base href="https://example.com/">

3. 🔤 文本与语义标签

🏷️ h1 ~ h6（标题） 基础

💡 一句话：六级标题，<h1> 级别最高，<h6> 最低。一个页面应该只有一个 <h1>。

标签默认字号语义权重使用建议
<h1>2em (32px)⭐⭐⭐⭐⭐⭐页面主标题，一个页面只用一次
<h2>1.5em (24px)⭐⭐⭐⭐⭐大章节标题
<h3>1.17em (19px)⭐⭐⭐⭐子章节标题
<h4>1em (16px)⭐⭐⭐子子标题
<h5>0.83em (13px)⭐⭐较少使用
<h6>0.67em (11px)⭐最少使用
🏷️ p / br / hr（段落与分隔） 基础
标签用途示例
<p>段落（Paragraph），块级，前后有间距<p>这是一段文字。</p>
<br>换行（Break），空元素第一行<br>第二行
<hr>水平分隔线（Horizontal Rule），空元素<hr>
<wbr>建议换行点（Word Break Opportunity）superlongword<wbr>breakhere
🏷️ 文本语义标签 基础

💡 一句话：用语义标签标记文本的含义，而不只是外观。搜索引擎和屏幕阅读器依赖它们。

标签语义默认样式使用场景
<strong>非常重要（Strong importance）粗体关键信息、警告
<em>强调（Emphasis），改变句子含义斜体"我真的不知道"vs"我真的不知道"
<b>引人注意，无语义重要性（Bring attention）粗体产品名、关键词
<i>另类语气（Idiomatic），如外来语、术语斜体书名、专业术语、外语短语
<u>非文本注解（Unarticulated annotation）下划线拼写错误标记（中文少用）
<s>不再正确/相关的内容（Strikethrough）删除线旧价格、已更正信息
<mark>高亮标记（Marked/Highlighted）黄色高亮搜索结果匹配词
<small>附属细则（Side comment）小号字版权声明、免责条款
<sub> / <sup>下标 / 上标H2O / E=mc2化学式、数学、脚注
<abbr>缩略词（Abbreviation），title 写全称虚线下方<abbr title="HyperText Markup Language">HTML</abbr>
<cite>引用作品标题斜体<cite>红楼梦</cite>
<q>行内引用，自动加引号引号<q>学而时习之</q>
<blockquote>块级引用缩进块大段引用内容，常用 cite 属性标来源 URL
<code>代码片段等宽字体<code>console.log()</code>
<pre>预格式化文本，保留空格和换行等宽+保留空白展示多行代码块
<kbd>键盘输入等宽按 <kbd>Ctrl+S</kbd>
<span>无语义行内容器无默认样式CSS/JS 操作目标（最通用的行内包装器）
<div>无语义块级容器块级，无默认样式布局容器（最通用的块级包装器）
💡
规则 of thumb： <b> vs <strong> — 如果去掉加粗也不影响内容理解，用 <b>；如果内容因此而变弱，用 <strong>。
<i> vs <em> — 如果只是样式斜体，用 CSS；如果是语气变化，用 <em>；如果是术语/外语，用 <i>。

4. 📋 列表与表格

🏷️ ul / ol / li / dl 基础

💡 一句话：三种列表 — 无序列表、有序列表、描述列表。可任意嵌套。

<!-- 无序列表 Unordered List -->
<ul>
  <li>苹果</li>
  <li>香蕉</li>
  <li>橘子</li>
</ul>

<!-- 有序列表 Ordered List -->
<ol>
  <li>第一步：准备食材</li>
  <li>第二步：下锅翻炒</li>
  <li>第三步：出锅装盘</li>
</ol>

<!-- 描述列表 Description List -->
<dl>
  <dt>HTML</dt>
  <dd>超文本标记语言，网页的结构层</dd>
  <dt>CSS</dt>
  <dd>层叠样式表，网页的表现层</dd>
</dl>

ol 的 type 属性（HTML5 仍支持，但推荐用 CSS list-style-type）

type 值编号样式
"1"（默认）1, 2, 3, ...
"A"A, B, C, ...
"a"a, b, c, ...
"I"I, II, III, ...
"i"i, ii, iii, ...
🏷️ table / thead / tbody / tfoot / tr / th / td 基础

💡 一句话：用 <table> 展示真正的表格数据（不要用表格做布局！）。

<table>
  <caption>2025年销售数据</caption>       <!-- 表格标题 -->
  <thead>                                    <!-- 表头 -->
    <tr>
      <th>月份</th><th>销售额</th><th>增长率</th>
    </tr>
  </thead>
  <tbody>                                    <!-- 表体 -->
    <tr>
      <td>1月</td><td>¥50,000</td><td>+5%</td>
    </tr>
    <tr>
      <td>2月</td><td>¥52,000</td><td>+4%</td>
    </tr>
  </tbody>
  <tfoot>                                    <!-- 表尾（汇总行） -->
    <tr>
      <td>合计</td><td>¥102,000</td><td>—</td>
    </tr>
  </tfoot>
</table>

单元格合并

属性含义示例
colspan="N"跨 N 列合并<td colspan="2">跨两列</td>
rowspan="N"跨 N 行合并<td rowspan="3">跨三行</td>

5. 🔗 链接与多媒体

🏷️ a（超链接） 基础
📐 语法:
<a href="URL">链接文字</a>

💡 一句话：HTML 的核心 — 超链接，连接互联网的基石。

🎬 场景⌨️ 代码📝 说明
外部链接<a href="https://example.com">访问网站</a>绝对 URL
内部页面<a href="/about.html">关于我们</a>相对于网站根目录
页内锚点<a href="#section2">跳到第二节</a>跳转到 id="section2" 的元素
新标签页打开<a href="url" target="_blank">务必加 rel="noopener noreferrer" 防安全漏洞
下载文件<a href="file.pdf" download>download 属性强制下载而非打开
邮件链接<a href="mailto:user@example.com">发邮件</a>打开默认邮件客户端
电话链接<a href="tel:+8613800000000">打电话</a>移动端拨号
⚠️
安全提醒：使用 target="_blank" 时，务必添加 rel="noopener noreferrer"。否则被打开的页面可以通过 window.opener 操纵你的页面（钓鱼攻击）。
🏷️ img（图片） 基础
📐 语法:
<img src="图片URL" alt="替代文字">

💡 一句话：空元素，alt 属性是必须的（无障碍访问 + 图片加载失败时的后备文字）。

属性必须?作用
src✅ 必须图片 URL
alt✅ 必须替代文字（屏幕阅读器朗读、SEO、加载失败显示）
width / height推荐预留空间，防止加载时的布局抖动（CLS）
srcset + sizes推荐响应式图片，不同屏幕加载不同分辨率
loading="lazy"推荐懒加载，图片进入视口前不加载
🏷️ video / audio（视频与音频） 基础

💡 一句话：HTML5 原生多媒体标签，无需 Flash 插件。

<!-- 视频 -->
<video controls width="640" poster="cover.jpg">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  您的浏览器不支持 video 标签。
</video>

<!-- 音频 -->
<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  您的浏览器不支持 audio 标签。
</audio>
常用属性作用
controls显示播放控件（播放/暂停/进度条）
autoplay自动播放（大多数浏览器需 + muted）
muted静音播放
loop循环播放
poster视频加载前显示的封面图（仅 video）
🏷️ iframe（内嵌框架） 进阶

💡 一句话：在一个页面中嵌入另一个独立页面。常用于嵌入视频、地图、第三方组件。

<iframe
  src="https://www.youtube.com/embed/VIDEO_ID"
  width="560" height="315"
  title="视频标题"
  frameborder="0"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope"
  allowfullscreen
  loading="lazy"
></iframe>
⚠️
安全：用 sandbox 属性限制 iframe 权限；始终设置 title（无障碍要求）。

6. 📝 表单与输入

🏷️ form（表单容器） 基础
📐 语法:
<form action="提交URL" method="GET|POST"> ... </form>

💡 一句话：<form> 是所有表单控件的容器，定义数据提交到哪里、怎么提交。

属性值说明
actionURL数据提交的目标地址
methodGET / POSTGET=URL参数（查询/搜索），POST=请求体（创建/修改数据）
enctypemultipart/form-data上传文件时必须设此值
novalidate布尔属性禁用浏览器的原生表单验证
🏷️ input（输入控件） 基础

💡 一句话：HTML 中最强大的表单元素，type 属性决定了它的形态和行为。

常用 type 值

type外观说明
text文本输入框默认值，单行文本
password密码框输入字符被遮盖
email邮箱输入框移动端弹出邮箱键盘，自动验证格式
number数字输入框增减按钮，min/max/step 约束
tel电话输入框移动端弹出数字键盘
urlURL 输入框自动验证 URL 格式
search搜索框带清除按钮（部分浏览器）
date日期选择器原生日期控件
time时间选择器原生时间控件
file文件选择accept 限制文件类型，multiple 允许多选
checkbox复选框多选，checked 表示默认选中
radio单选按钮同名 name 的为一组，互斥
hidden隐藏字段用户不可见但随表单提交
submit提交按钮value 属性设置按钮文字
reset重置按钮清空表单所有输入
range滑块min/max/step 定义范围
color颜色选择器原生颜色拾取器

常用通用属性

属性作用
name表单提交时的键名（必须，否则该字段不会被提交）
value默认值 / 提交的值
placeholder输入提示文字（灰色占位符）
required必填项，提交前验证
disabled禁用，不可交互也不会被提交
readonly只读，可选中但仍会被提交
autofocus页面加载后自动聚焦
pattern正则表达式验证（如 pattern="[0-9]{6}"）
🏷️ select / textarea / button / label / fieldset 基础 
<!-- 下拉选择 -->
<label for="city">城市：</label>
<select id="city" name="city">
  <optgroup label="华东">
    <option value="sh">上海</option>
    <option value="hz">杭州</option>
  </optgroup>
  <optgroup label="华北">
    <option value="bj" selected>北京</option>
  </optgroup>
</select>

<!-- 多行文本 -->
<label for="bio">个人简介：</label>
<textarea id="bio" name="bio" rows="5" cols="40" placeholder="请输入..."></textarea>

<!-- 按钮 -->
<button type="submit">提交</button>
<button type="reset">重置</button>
<button type="button">普通按钮（JS 驱动）</button>

<!-- 字段分组 -->
<fieldset>
  <legend>个人信息</legend>
  <!-- 表单控件 -->
</fieldset>
💡
label 的重要性：每个表单控件都应有 <label> 关联（通过 for 匹配 id，或把控件包在 label 内）。点击 label 文字会自动聚焦控件，屏幕阅读器也依赖它。

7. 🏗️ HTML5 语义化

HTML5 引入了一组语义化结构标签，替代了之前泛滥的 <div class="header">、<div class="nav"> 等写法。它们让页面结构更清晰，对 SEO 和屏幕阅读器更友好。

7.1 典型页面结构

graph TD BODY["<body>"] --> HEADER["<header>
页头：Logo + 导航"] BODY --> NAV["<nav>
主导航菜单"] BODY --> MAIN["<main>
页面主体（唯一）"] BODY --> FOOTER["<footer>
页脚：版权、链接"] MAIN --> ARTICLE["<article>
独立内容块"] MAIN --> SECTION["<section>
主题分组"] MAIN --> ASIDE["<aside>
侧边栏/补充"] ARTICLE --> H1_ART["<h1>"] ARTICLE --> P_ART["<p>"]

7.2 语义标签速查

标签语义使用场景
<header>页头 / 区块头顶部导航、Logo；也可出现多次（如 article header）
<nav>导航链接区块主导航、面包屑、分页（非所有链接组都用 nav）
<main>页面主要内容⭐ 一个页面只能用 一个 main
<article>独立的内容块博客文章、新闻、评论、产品卡片（可独立分发）
<section>主题分组有标题的内容区域（没有标题就别用 section）
<aside>侧边栏 / 补充内容侧边栏、广告、相关链接、注释
<footer>页脚 / 区块脚版权信息、联系链接、相关文档
<figure> + <figcaption>插图 + 标题图表、照片、代码块等带说明的独立内容
<time>日期/时间<time datetime="2025-07-10">2025年7月10日</time>
<address>联系信息作者邮箱、地址、电话（仅用于联系信息）
<details> + <summary>可折叠面板FAQ、展开/收起内容，无需 JS
<dialog>对话框模态框、确认弹窗（需 JS 控制 open）
<progress>进度条<progress value="70" max="100">70%</progress>
<meter>度量衡磁盘使用率、评分（有范围的量）
💡
div 还能用吗？当然！当没有任何语义标签适合时，<div> 和 <span> 仍然是最佳的通用容器。不要为了用语义标签而强行套用。

8. 🏷️ 属性与全局属性

🎯 全局属性（可用于几乎所有元素） 基础
属性值作用
id唯一标识符页面内唯一；CSS 选择器、JS 查找、锚点目标
class空格分隔的类名列表CSS 样式、JS 批量操作（最常用的属性）
styleCSS 声明行内样式（尽量少用，用外部 CSS）
title提示文字鼠标悬停时显示的提示（tooltip）
lang语言代码元素内容的语言（en, zh-CN, ja）
dirltr / rtl / auto文字方向（左到右 / 右到左）
hidden布尔属性隐藏元素（CSS display:none 等效）
tabindex整数Tab 键导航顺序（-1=可编程聚焦但不可 tab）
contenteditabletrue / false元素是否可编辑
draggabletrue / false是否可拖拽
accesskey单个字符键盘快捷键（不同浏览器触发方式不同，少用）
🎯 data-* 自定义属性 / aria-* 无障碍属性 进阶

data-* 自定义数据属性

<div data-user-id="123" data-role="admin">
  <!-- JS 中通过 dataset 访问 -->
</div>

<script>
  const div = document.querySelector('div');
  console.log(div.dataset.userId);   // "123"（驼峰命名）
  console.log(div.dataset.role);      // "admin"
</script>

aria-* 无障碍属性（WAI-ARIA）

属性作用示例
aria-label给无文字元素提供可读标签<button aria-label="关闭">×</button>
aria-hidden对屏幕阅读器隐藏纯装饰性图标
aria-expanded表示展开/折叠状态汉堡菜单、手风琴
aria-live动态内容区域，自动播报变化聊天消息、通知、搜索建议
role定义元素角色role="navigation"、role="button"

9. 🎨 CSS/JS 集成

🏷️ style / link / script 基础

💡 一句话：HTML 与 CSS/JavaScript 协同工作的三种方式。

CSS 集成（三种方式，优先级：行内 > 内部 > 外部）

<!-- ① 外部样式表（推荐！分离关注点）-->
<link rel="stylesheet" href="style.css">

<!-- ② 内部样式（在 head 中）-->
<style>
  body { font-family: sans-serif; }
</style>

<!-- ③ 行内样式（尽量少用）-->
<p style="color: red;">红色文字</p>

JavaScript 集成

<!-- 外部脚本（推荐放 body 末尾）-->
<script src="app.js"></script>

<!-- 内部脚本 -->
<script>
  console.log('Hello World');
</script>

<!-- 脚本加载策略 -->
<script src="heavy.js" defer></script>   <!-- HTML解析完后执行，保持顺序 -->
<script src="third.js" async></script>   <!-- 下载完立刻执行，不保证顺序 -->

<!-- 旧式写法：type="module" 自动 defer -->
<script type="module" src="app.js"></script>
属性行为适用场景
（无）暂停 HTML 解析，下载并执行 JS，然后继续❌ 阻塞渲染，不推荐
defer并行下载，HTML 解析完成后按顺序执行⭐ 大多数情况的最佳选择
async并行下载，下载完立刻执行（不保证顺序）独立第三方脚本（统计、广告）
type="module"自动 defer，支持 import/export现代 ES Module 项目

10. 📱 响应式与SEO

🎯 响应式图片与 viewport 进阶

viewport（移动端必须）

<meta name="viewport" content="width=device-width, initial-scale=1.0">

srcset + sizes（响应式图片）

<!-- 根据屏幕宽度加载不同尺寸图片 -->
<img
  src="photo-800.jpg"
  srcset="photo-400.jpg 400w, photo-800.jpg 800w, photo-1200.jpg 1200w"
  sizes="(max-width: 600px) 100vw, (max-width: 1200px) 50vw, 33vw"
  alt="风景照片"
>

picture（艺术方向 — 不同场景加载不同图片）

<picture>
  <source media="(max-width: 600px)" srcset="hero-mobile.jpg">
  <source media="(max-width: 1200px)" srcset="hero-tablet.jpg">
  <img src="hero-desktop.jpg" alt="Hero 图片">
</picture>
🎯 SEO 与社交分享标签 进阶

💡 一句话：让搜索引擎更好地理解你的页面，让社交平台生成漂亮的分享卡片。

基础 SEO meta 标签

<title>页面标题 — 网站名</title>
<meta name="description" content="150字内的页面描述，搜索结果中显示">
<link rel="canonical" href="https://example.com/page">  <!-- 规范URL，防重复内容 -->
<meta name="robots" content="index, follow">           <!-- 允许收录和跟踪链接 -->

Open Graph（社交分享卡片 — Facebook、Twitter、微信等）

<meta property="og:title" content="页面标题">
<meta property="og:description" content="分享描述">
<meta property="og:image" content="https://example.com/og-image.jpg">
<meta property="og:url" content="https://example.com/page">
<meta property="og:type" content="website">

<!-- Twitter 专用 -->
<meta name="twitter:card" content="summary_large_image">

结构化数据（JSON-LD — 搜索引擎增强展示）

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "文章标题",
  "author": { "@type": "Person", "name": "作者名" },
  "datePublished": "2025-07-10"
}
</script>
