1. 🗺️ 核心概念速览

1.1 PyTorch 是什么

PyTorch 是当前最流行的深度学习框架之一，由 Meta AI 开发维护。核心特点：
• 动态计算图（Define-by-Run）— 每次前向传播都可以构建不同的图，调试友好
• Pythonic — 与 NumPy 高度相似，无缝融入 Python 生态
• 强大的 GPU 加速 — 一行代码在 CPU/GPU 间切换
• 丰富的生态系统 — torchvision, torchaudio, torchtext, HuggingFace, PyTorch Lightning

1.3 动态计算图 vs 静态计算图

特性PyTorch（动态图）TensorFlow 1.x（静态图）
构建方式每步即时执行，随写随运行（Eager Mode）先定义完整图，再统一执行
调试✅ 可用 Python 调试器、print 任意位置❌ 难以调试，图构建和执行分离
条件/循环✅ 原生 Python if/for，灵活需要特殊控制流 API（tf.cond, tf.while）
性能较优（torch.compile 正在追赶）优化空间更大
部署TorchScript / ONNX / torch.exportSavedModel / TFLite

1.4 最小训练代码骨架

import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader, TensorDataset

# 1. 数据
X = torch.randn(1000, 10)
y = torch.randint(0, 2, (1000,))
loader = DataLoader(TensorDataset(X, y), batch_size=32, shuffle=True)

# 2. 模型
model = nn.Sequential(nn.Linear(10, 64), nn.ReLU(), nn.Linear(64, 2))

# 3. 损失 + 优化器
criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=1e-3)

# 4. 训练循环
for epoch in range(10):
    for xb, yb in loader:
        pred = model(xb)               # 前向传播
        loss = criterion(pred, yb)      # 计算损失
        optimizer.zero_grad()           # 梯度清零
        loss.backward()                 # 反向传播
        optimizer.step()                # 更新参数
    print(f"Epoch {epoch+1}, Loss: {loss.item():.4f}")

3. 📐 自动求导 (Autograd)

PyTorch 的自动求导引擎是所有神经网络训练的基石。它在后台构建计算图，backward() 时自动计算所有参数的梯度。
🎯 梯度追踪与计算 核心
🎬 场景⌨️ 代码📝 说明
开启梯度追踪x = torch.tensor([2.0], requires_grad=True)创建时指定
稍后开启x.requires_grad_(True)下划线后缀 = in-place 修改
反向传播loss.backward()标量调用，计算所有 requires_grad=True 的梯度
获取梯度x.grad计算后的梯度值
梯度清零optimizer.zero_grad()⭐ 每个训练 step 前必须清零
手动清零x.grad.zero_()手动清零某个张量的梯度
禁止梯度追踪with torch.no_grad():⭐ 推理/评估时必须用，省显存+加速
分离计算图x.detach()返回一个新张量，与计算图断开
梯度裁剪torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)防止梯度爆炸（RNN 常用）
计算梯度范数torch.nn.utils.clip_grad_norm_ 不裁剪只计算监控梯度是否正常
🎯 关键概念详解 进阶

requires_grad 传播规则

a = torch.randn(3, requires_grad=True)
b = a * 2           # b.requires_grad → True（继承）
c = b.detach()      # c.requires_grad → False（断开）
d = c + 1           # d.requires_grad → False

no_grad vs eval()

操作作用何时用
torch.no_grad()禁用 autograd 图构建推理、验证、评估（省显存）
model.eval()设置模型为评估模式影响 Dropout（关闭）、BatchNorm（用全局统计）
model.train()设置模型为训练模式恢复正常训练行为
⚠️
常见陷阱：验证/推理时两个都要用 — model.eval() 切换行为，torch.no_grad() 关闭梯度记录。只用一个是不够的！

自定义 Dataset（模板）

from torch.utils.data import Dataset

class MyDataset(Dataset):
    def __init__(self, data_dir, transform=None):
        self.data = ...                         # 加载文件列表或数据

5. 🧠 网络层 nn.Module

🎯 nn.Module 模板 核心

💡 一句话：所有神经网络的基类。__init__ 中定义层，forward 中定义前向逻辑。

import torch.nn as nn
import torch.nn.functional as F

class MyNet(nn.Module):
    def __init__(self, in_dim, hidden_dim, num_classes):
        super().__init__()
        self.fc1 = nn.Linear(in_dim, hidden_dim)
        self.bn = nn.BatchNorm1d(hidden_dim)
        self.dropout = nn.Dropout(0.3)
        self.fc2 = nn.Linear(hidden_dim, num_classes)

    def forward(self, x):
        x = self.fc1(x)
        x = self.bn(x)
        x = F.relu(x)
        x = self.dropout(x)
        x = self.fc2(x)
        return x

model = MyNet(784, 256, 10)
print(model)  # 打印完整结构
print(f"参数量: {sum(p.numel() for p in model.parameters()):,}")
🎯 常用层速查 核心
分类层核心参数
全连接nn.Linear(in, out)输入/输出维度
nn.Identity()占位层（不做任何转换）
卷积nn.Conv2d(in_c, out_c, k, s, p)输入/输出通道、卷积核尺寸、步长、填充
nn.ConvTranspose2d(...)转置卷积（上采样）
nn.MaxPool2d(k) / nn.AdaptiveAvgPool2d((1,1))池化（后者自动适配到目标尺寸）

6. 🎯 损失函数与优化器

🎯 常用损失函数 核心
损失函数适用场景说明
nn.CrossEntropyLoss()多分类（⭐ 最常用）内置 softmax + NLLLoss，输入 raw logits
nn.BCEWithLogitsLoss()二分类 / 多标签⭐ 推荐（比 BCE + Sigmoid 分开写数值更稳）
nn.MSELoss()回归均方误差
nn.L1Loss()回归（鲁棒）MAE，对异常值不敏感
nn.HuberLoss()回归（平滑）L1 和 L2 的折中
nn.NLLLoss()分类（配 log_softmax）通常直接用 CrossEntropyLoss
nn.KLDivLoss()KL 散度（蒸馏）知识蒸馏常用
nn.CosineEmbeddingLoss()相似度学习度量学习 / 对比学习
nn.TripletMarginLoss()三元组损失人脸识别、检索
⚠️
CrossEntropyLoss 常见错误：
• 输入应为 raw logits（不要先做 softmax）— CrossEntropyLoss 内部已经包含 softmax
• labels 应为 类索引整数（不是 one-hot）— torch.tensor([2, 5, 1]) 而非 one-hot 向量
• 缺少 ignore_index 参数可能导致 padding 位置被错误计算
🎯 常用优化器 核心
优化器推荐场景典型参数
optim.Adam(params, lr=1e-3)⭐ 通用首选betas=(0.9, 0.999), weight_decay=1e-4
optim.AdamW(params, lr=1e-3)⭐ Transformer / 大模型Adam + 解耦权重衰减（更正确的正则化）
optim.SGD(params, lr=0.01, momentum=0.9)图像分类（ResNet 等）配合 scheduler 效果最佳
optim.RMSprop(params, lr=1e-3)RNN / 强化学习alpha=0.99
 
# ⭐ 推荐：带权重衰减的 AdamW
optimizer = optim.AdamW(model.parameters(), lr=1e-3, weight_decay=0.01)

# 分组设置不同学习率
optimizer = optim.AdamW([
    {'params': model.backbone.parameters(), 'lr': 1e-4},    # backbone 低学习率
    {'params': model.head.parameters(),    'lr': 1e-3},    # head 高学习率
], weight_decay=0.01)
🎯 学习率调度器 (lr_scheduler) 进阶
调度器行为适用场景
StepLR(opt, step_size=30, gamma=0.1)每 N 个 epoch 将 lr 乘以 gamma基准调度
CosineAnnealingLR(opt, T_max=100)余弦曲线平滑降至 0⭐ 常用
ReduceLROnPlateau(opt, mode='min', patience=5)指标不降时降低 lr自适应
OneCycleLR(opt, max_lr=0.01, steps_per_epoch=...)先升后降，快速收敛⭐ 训练加速
CosineAnnealingWarmRestarts(opt, T_0=10)余弦退火 + 周期性重启训练更久

    train_one_epoch(...)

7. 🏋️ 训练与评估

🎯 标准训练模板 核心 
def train_epoch(model, loader, criterion, optimizer, device):
    model.train()
    total_loss, correct = 0, 0
    for x, y in loader:
        x, y = x.to(device), y.to(device)

        optimizer.zero_grad()
        output = model(x)
        loss = criterion(output, y)
        loss.backward()
        optimizer.step()

        total_loss += loss.item() * x.size(0)
        correct += (output.argmax(1) == y).sum().item()

    return total_loss / len(loader.dataset), correct / len(loader.dataset)

@torch.no_grad()
def evaluate(model, loader, criterion, device):
    model.eval()
    total_loss, correct = 0, 0
    for x, y in loader:
        x, y = x.to(device), y.to(device)
        output = model(x)
        loss = criterion(output, y)
        total_loss += loss.item() * x.size(0)
        correct += (output.argmax(1) == y).sum().item()
    return total_loss / len(loader.dataset), correct / len(loader.dataset)

🎯 Early Stopping 模板 进阶 
class EarlyStopping:
    def __init__(self, patience=10, min_delta=0, mode='min'):
        self.patience = patience
        self.min_delta = min_delta
        self.mode = mode
        self.counter = 0
        self.best_score = float('inf') if mode == 'min' else float('-inf')
        self.early_stop = False

    def __call__(self, score):
        if self.mode == 'min':
            improved = score < self.best_score - self.min_delta
        else:
            improved = score > self.best_score + self.min_delta

        if improved:
            self.best_score = score
            self.counter = 0
        else:
            self.counter += 1
            if self.counter >= self.patience:
                self.early_stop = True
        return self.early_stop

🎯 混合精度训练 (AMP) 进阶

💡 一句话：用 float16 做前向/反向（快+省显存），关键步骤用 float32 保精度。几乎无代价提速 30-50%。

scaler = torch.amp.GradScaler()   # 防止 float16 梯度下溢

for x, y in loader:
    x, y = x.to(device), y.to(device)
    optimizer.zero_grad()

    with torch.amp.autocast(device_type='cuda'):  # 自动选择 float16/float32
        output = model(x)
        loss = criterion(output, y)

    scaler.scale(loss).backward()   # loss 放大 → 反向 → 梯度缩回
    scaler.step(optimizer)
    scaler.update()

🎯 迁移学习（Transfer Learning） 核心

💡 一句话：拿 ImageNet 预训练的 backbone，替换最后的分类头，在新数据上微调。这是业界最常用的模式。

    param.requires_grad = False

# 替换最后的全连接层

# 只用新数据训练 model.fc

# 或者：解冻全部，用不同的学习率

    {'params': model.conv1.parameters(), 'lr': 1e-5},
    {'params': model.layer1.parameters(), 'lr': 1e-5},
    {'params': model.layer2.parameters(), 'lr': 1e-4},
    {'params': model.layer3.parameters(), 'lr': 1e-4},
    {'params': model.layer4.parameters(), 'lr': 1e-4},
    {'params': model.fc.parameters(),     'lr': 1e-3},  # head 用高 lr

🎯 梯度累积（模拟大 Batch Size） 进阶

💡 一句话：显存不够设大 batch_size 时，通过多次小 batch 累积梯度来模拟大 batch 的效果。

accumulation_steps = 4   # 模拟 4×batch_size

for i, (x, y) in enumerate(loader):
    x, y = x.to(device), y.to(device)
    output = model(x)
    loss = criterion(output, y) / accumulation_steps  # ⚠️ 除以累积步数
    loss.backward()                                     # 梯度累加，不清零

    if (i + 1) % accumulation_steps == 0:
        optimizer.step()     # 每 accumulation_steps 次才更新
        optimizer.zero_grad()
