1. 🗺️ 核心概念速览

1.1 Linux 文件系统层次结构

Linux 的一切皆文件，整个系统从根目录 / 开始，形成一个倒挂的树状结构。

启动加载器文件"] R --> DEV["/dev

共享库文件"] R --> MEDIA["/media

临时文件"] R --> USR["/usr

可变数据(日志/缓存等)"]

1.2 关键目录速查

目录用途记忆方法
/etc配置文件集中地（类似 Windows 注册表）"Edit Things Config"
/var日志、缓存、数据库等可变数据"Variable data"
/tmp临时文件，重启后清空"Temporary"
/usr用户级程序和库（非系统核心）"Unix System Resources"
/opt第三方大型软件（如 Google Chrome）"Optional"
/proc / /sys虚拟文件系统，内核暴露的运行时信息"Process" / "System"
/dev设备文件（硬盘、终端、USB 等）"Devices"

1.3 Shell 与终端

终端 (Terminal) 是一个窗口/容器；Shell 是运行在终端里的命令解释器。
常见 Shell：bash（默认）、zsh（功能更丰富）、fish（开箱即用）、dash（轻量 POSIX）。

Shell 提示符解读

user@hostname:~$
部分含义
user当前登录用户名
@分隔符，"在"的意思
hostname主机名
:分隔符
~当前目录（~ = 用户主目录，如 /home/user）
$普通用户提示符（# 表示 root 超级用户）

1.4 绝对路径 vs 相对路径

类型特征示例
绝对路径从 / 开始的完整路径/home/user/docs/file.txt
相对路径相对于当前目录的路径./docs/file.txt 或 ../docs/file.txt
特殊符号含义
.当前目录
..上一级目录
~当前用户主目录
~useruser 用户的主目录
-上一次所在目录（cd - 切换回去）

2. 📁 文件与目录篇

🎯 ls 入门 只读
📐 公式:
ls [选项] [路径]

💡 一句话：列出目录内容，Linux 中使用频率最高的命令之一。

🎬 场景⌨️ 命令📝 说明
基本列表ls列出当前目录文件和文件夹

递归列出ls -R递归列出子目录内容
只列目录ls -d */只显示目录项
最常用组合ls -lah长格式+隐藏文件+人类可读 = 日常首选

💡 一句话：切换当前工作目录（Change Directory）。

🎬 场景⌨️ 命令📝 说明
去某目录cd /path/to/dir切换到指定绝对路径
回主目录cd 或 cd ~快速回到用户主目录
回上一级cd ..返回父目录
回上两级cd ../..返回父目录的父目录
回上一次目录cd -切换到上次所在目录（反复执行在两个目录间切换）

💡 一句话：创建新目录（Make Directory）。

🎬 场景⌨️ 命令📝 说明
创建目录mkdir mydir创建单个目录
递归创建mkdir -p a/b/c/d自动创建所有不存在的父目录（最实用选项）

🎯 cp 注意
📐 公式:
cp [选项] 源 目标

💡 一句话：复制文件或目录（Copy）。

🎬 场景⌨️ 命令📝 说明
复制文件cp file.txt backup.txt复制文件并重命名
复制到目录cp file.txt /target/dir/保持原名复制到目标目录
递归复制目录cp -r src_dir/ dst_dir/复制整个目录树（最常用选项）
保留属性cp -rp src/ dst/保留权限、时间戳等元信息
交互确认cp -i file.txt dst/覆盖前询问确认
强制覆盖cp -f file.txt dst/不询问直接覆盖
⚠️
注意：cp 默认不保留原始权限和时间戳，重要备份请加 -p（preserve）。
🎯 mv 注意
📐 公式:
mv [选项] 源 目标

💡 一句话：移动/重命名文件或目录（Move）。

🎬 场景⌨️ 命令📝 说明
重命名mv old.txt new.txt同一目录下 = 重命名
移动文件mv file.txt /target/dir/将文件移到目标目录
移动目录mv dir1/ /target/移动整个目录（无需 -r，直接移）
交互确认mv -i source target覆盖前询问
不覆盖mv -n source target目标存在则跳过（no-clobber）
🎯 rm 危险
📐 公式:
rm [选项] 文件/目录

💡 一句话：删除文件或目录（Remove），Linux 中没有回收站！

🎬 场景⌨️ 命令📝 说明
删除文件rm file.txt删除单个文件
删除多个文件rm file1.txt file2.txt一次删除多个
递归删除目录rm -r dir/删除目录及其所有内容
强制删除rm -f file.txt不提示确认，忽略不存在的文件
强制递归删除rm -rf dir/⚠️ 最危险组合，不询问直接删光目录树
交互确认rm -i file.txt每个文件删除前需手动确认
详细输出rm -v *.log显示每个被删除的文件名
🔴
危险警告：
• rm -rf / 会尝试删除整个系统（现代系统有保护，但别试）
• rm -rf * 删除当前目录所有内容
• rm -rf .* 包含 .. 会波及其父目录
• 建议：重要操作前先用 ls 确认目标，或用 rm -i 分批确认
🎯 find 进阶 只读
📐 公式:
find [路径] [匹配条件] [动作]

💡 一句话：强大的文件搜索工具，按名称、类型、大小、时间等条件查找文件。

🎬 场景⌨️ 命令📝 说明
按名称查找find . -name "*.log"在当前目录及子目录中查找 .log 文件
忽略大小写find . -iname "readme*"不区分大小写匹配
按类型查找find . -type f只找普通文件（d=目录, l=符号链接）
按大小查找find . -size +100M找大于 100MB 的大文件
按时间查找find . -mtime -7最近 7 天内修改过的文件
查找并执行find . -name "*.tmp" -delete查找并删除（也可用 -exec rm {} \;）
查找空文件find . -empty查找空文件或空目录
限制深度find . -maxdepth 2 -name "*.js"只搜索当前目录及下一层子目录
🎯 ln 进阶
📐 公式:
ln [选项] 源文件 链接名

💡 一句话：创建文件链接（Link），分硬链接和软链接（符号链接）。

🎬 场景⌨️ 命令📝 说明
创建软链接ln -s /path/to/original link符号链接（类似 Windows 快捷方式），最常用
创建硬链接ln original hardlink硬链接（同一文件多入口，共享 inode）
强制覆盖ln -sf original link强制重新创建符号链接
特性硬链接软链接（符号链接）
跨文件系统❌ 不可以✅ 可以
链接目录❌ 不可以✅ 可以
源删除后数据仍在（inode 引用计数>0）链接断裂（悬空链接）
查看方式ls -l 第二列 = 硬链接数ls -l 显示箭头 → 指向源

📐 公式:

💡 一句话：文本搜索神器，在文件或管道输入中查找匹配行（Global Regular Expression Print）。

🎬 场景⌨️ 命令📝 说明
基本搜索grep "error" log.txt在文件中搜索含 error 的行

4. 🔐 权限管理篇

4.1 权限模型解读

Linux 权限基于 UGO 模型（User / Group / Other），每个文件有三组权限：

4.2 八进制权限对照

八进制二进制权限含义
0000---无权限
1001--x仅执行

🎯 kill / pkill / killall 注意
📐 公式:
kill [信号] PID
pkill [选项] 模式
killall [选项] 进程名

💡 一句话：向进程发送信号，终止或控制进程行为。

常用信号

编号名称作用能否被捕获/忽略
1SIGHUP挂起（常用于重新加载配置）✅ 可以
2SIGINT中断（相当于 Ctrl+C）✅ 可以
3SIGQUIT退出（相当于 Ctrl+\）✅ 可以
9SIGKILL强制杀死（不给清理机会）❌ 不可捕获
15SIGTERM优雅终止（默认信号）✅ 可以
18SIGCONT恢复暂停的进程—
19SIGSTOP暂停进程（不可被忽略）❌ 不可捕获
🎬 场景⌨️ 命令📝 说明
优雅终止kill 1234发送 TERM 信号，给进程清理机会
强制杀死kill -9 1234⚠️ 立刻终止，不给清理机会（最后手段）
重载配置kill -HUP 1234很多守护进程收到 HUP 会重新读取配置
按名杀进程pkill nginx杀死所有名含 nginx 的进程
模糊匹配杀pkill -f "python app.py"匹配完整命令行
交互确认pkill -i nginx每个匹配进程询问确认
按用户名杀pkill -u alice杀死 alice 用户的所有进程
精确名杀killall nginx按精确进程名杀死（小心同名）
⚠️
原则：先 kill (TERM) → 不行再 kill -2 (INT) → 最后才 kill -9 (KILL)。-9 不给进程清理资源的机会，可能导致数据丢失。

dd if=输入 of=输出 [选项]

💡 一句话：lsblk 查看块设备，fdisk 分区管理，dd 低级数据复制。

🎬 场景⌨️ 命令📝 说明
查看块设备lsblk树状显示所有块设备及挂载点
查看磁盘列表sudo fdisk -l列出所有磁盘和分区表
分区操作sudo fdisk /dev/sdb交互式分区
制作启动U盘sudo dd if=ubuntu.iso of=/dev/sdb bs=4M status=progress⚠️ of 目标会被全部覆写！
磁盘克隆sudo dd if=/dev/sda of=/dev/sdb bs=4M⚠️ 整盘克隆，目标盘所有数据丢失！
擦除磁盘sudo dd if=/dev/zero of=/dev/sdb bs=1M⚠️ 用零填充整个磁盘（不可恢复）
🔴
dd = Disk Destroyer：dd 的命令行参数没有确认提示，if 和 of 写反会立刻毁灭目标数据。
执行前务必再三确认 of= 是否正确。用 lsblk 先看清楚设备名。

10.3 条件判断

else
    echo "$a < $b"
fi

# 现代写法 [[ ]]（bash 专有，更安全）
if [[ "$name" == "admin" && "$age" -ge 18 ]]; then
    echo "Access granted"
fi

字符串含义数字含义文件含义

-z为空-gt大于-e存在
-n非空-lt小于-x可执行
