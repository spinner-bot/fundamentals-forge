1. 🗺️ 核心概念速览

1.1 什么是 LaTeX

LaTeX 不是一个"所见即所得"的文字处理器，而是一个基于命令的排版系统。
你写的是带标记的纯文本（.tex 文件），编译器将它渲染成精美的 PDF。
类比：LaTeX 之于文档，如同 HTML+CSS 之于网页 — 你定义结构和内容，它负责排版输出。

⚠️
重要：LaTeX 通常需要编译多次才能正确生成交叉引用、目录和参考文献。
推荐流程：xelatex → bibtex/biber → xelatex → xelatex（共 2~4 次）。
工具 latexmk 可以自动处理编译次数。

2. 📐 文档基础结构

2.1 最小可编译文档

% hello.tex — 最小 LaTeX 文档
\documentclass{article}  % ① 文档类型

% 导言区：加载宏包、设置参数
\usepackage{ctex}        % 中文支持

\title{我的第一篇文档}
\author{张三}
\date{\today}

\begin{document}         % ② 正文开始
\maketitle               % ③ 生成标题

\section{引言}
这是正文内容。

\section{结论}
\end{document}           % ④ 正文结束（之后的内容被忽略）

2.2 文档结构四要素

部分位置作用
① 文档类声明第 1 行决定文档基础样式：article / report / book / ctexart / beamer
② 导言区class 之后，begin{document} 之前加载宏包、定义命令、设置参数
③ 正文区begin{document} ~ end{document}所有可见内容和结构命令
④ 注释% 之后到行尾编译时被忽略，给自己看
📐 \\documentclass 基础
📐 语法:
\documentclass[选项]{文档类}

💡 一句话：声明文档类型，决定整体排版风格和可用命令。

常用文档类

文档类用途特点
article短文、论文无 chapter，顶层是 section
report报告、学位论文有 chapter，每章新起一页
book书籍有 chapter + part，双面排版
ctexart中文短文中文字体、间距自动配置
ctexrep中文报告中文 book 的替代
beamer演示文稿（PPT）生成 PDF 幻灯片
letter信件信件格式

常用选项（英文逗号分隔）

选项含义
12pt正文字号 12pt（默认 10pt）
a4paperA4 纸张（默认 letterpaper）
twocolumn双栏排版
twoside双面打印（奇偶页不同）
draft草稿模式（图片不加载，标记过满行）
📐 章节层次命令 基础
📐 语法:
\section{标题}
\section[短标题]{长标题}

💡 一句话：文档的层次化结构，LaTeX 自动编号和排版。

层次从高到低

命令层次可用于说明
\part{}-1book, report部，最高层次
\chapter{}0book, report章
\section{}1全部节
\subsection{}2全部子节
\subsubsection{}3全部子子节
\paragraph{}4全部段落标题

带 * 的版本（如 \section*{}）不编号，也不出现在目录中。

📐 标题信息与 maketitle 基础
📐 语法:
\title{标题}
\author{作者}
\date{日期}
\maketitle

💡 一句话：定义文档标题信息并在正文中生成标题区域。

\date{} 留空则不显示日期；不写则自动打印 \today。多个作者用 \and 分隔。

小大写\textsc{文字}{\scshape 文字}
无衬线\textsf{文字}{\sffamily 文字}
强调\emph{文字}—（嵌套自动切换 upright/italic）
下划线\underline{文字}—

🎯 段落与间距 基础

💡 一句话：LaTeX 中空行即分段，空格和缩进有精细的控制命令。

🎯 列表环境 基础

💡 一句话：三种标准列表环境 — 无序列表 (itemize)、有序列表 (enumerate)、描述列表 (description)。

% 无序列表
\begin{itemize}
  \item 项目一
  \item 项目二
  \begin{itemize}
    \item 嵌套子项
  \end{itemize}
\end{itemize}

% 有序列表（自动编号）
\begin{enumerate}
  \item 第一步
  \item 第二步
\end{enumerate}

% 描述列表（自定义标签）
\begin{description}
  \item[LaTeX] 一个排版系统
  \item[Markdown] 轻量标记语言
\end{description}

使用 enumitem 宏包可以获得更强大的列表定制能力（修改标签样式、间距等）。

🎯 对齐环境 基础

💡 一句话：控制文本的对齐方式。

🎬 场景⌨️ 命令

5. 📊 表格

🎯 tabular 环境 基础
📐 语法:
\begin{tabular}{列格式}
~~内容
\end{tabular}

💡 一句话：LaTeX 的核心表格环境，& 分隔列，\\ 分隔行。

列格式说明符

符号含义
l左对齐列（Left）
c居中列（Center）
r右对齐列（Right）
p{宽度}固定宽度段落列（可自动换行）
|竖线（可多次使用：|| 双竖线）
 
% 基本三列表格
\begin{tabular}{l|c|r}
  左对齐 & 居中 & 右对齐 \\
  \hline                      % 水平线
  A      & B    & C    \\
  D      & E    & F
\end{tabular}
🎯 三线表（booktabs 宏包） 进阶

💡 一句话：学术论文中最推荐的表格式 — 只有顶线、表头底线、底线三条线，简洁专业。

\usepackage{booktabs}

\begin{tabular}{lcc}
  \toprule                        % 顶线（粗）
  项目 & 数值A & 数值B \\
  \midrule                        % 中线（细）
  X    & 1.23  & 4.56 \\
  Y    & 7.89  & 0.12 \\
  \bottomrule                     % 底线（粗）
\end{tabular}
命令作用
\toprule表格顶部粗线
\midrule表头与数据之间的细线
\bottomrule表格底部粗线
\cmidrule{2-3}跨列的部分横线（第 2~3 列下方）
🎯 table 浮动体 / 合并单元格 进阶

💡 一句话：table 环境让表格成为可自动定位、可加标题和标签的浮动体；合并单元格用 \multicolumn 和 \multirow。

% table 浮动体（自动定位，可加标题）
\begin{table}[htbp]           % h=here, t=top, b=bottom, p=page
  \centering
  \caption{表格标题}
  \label{tab:mytable}
  \begin{tabular}{lcr}
    ...
  \end{tabular}
\end{table}

% 合并列
\multicolumn{2}{c}{跨两列居中文字}

% 合并行（需 multirow 宏包）
\usepackage{multirow}
\multirow{2}{*}{跨两行文字}

[htbp] 位置选项：h=当前位置，t=页顶，b=页底，p=独立页，!=强制尝试（如 [!htbp]）。

6. 🖼️ 图片与浮动体

🎯 graphicx 宏包 基础
📐 语法:
\usepackage{graphicx}
\includegraphics[选项]{图片文件名}

💡 一句话：插图的标准宏包，支持 PDF/PNG/JPG/EPS 等多种格式。

🎬 场景⌨️ 命令📝 说明
基本插图\includegraphics{fig.pdf}原尺寸插入
按比例缩放\includegraphics[scale=0.5]{fig}缩放到 50%
指定宽度\includegraphics[width=0.8\textwidth]{fig}宽度为正文宽度的 80%
指定高度\includegraphics[height=5cm]{fig}高度 5cm，宽度按比例
旋转\includegraphics[angle=90]{fig}逆时针旋转 90°
裁剪\includegraphics[trim=1cm 2cm 1cm 2cm, clip]{fig}裁剪四边（左 下 右 上）

图片路径设置

\graphicspath{{figures/}{images/}}  % 指定图片搜索目录（导言区）
🎯 figure 浮动体 基础

💡 一句话：figure 环境让图片自动寻找最佳位置，可加标题和交叉引用。

\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.6\textwidth]{diagram.pdf}
  \caption{系统架构图}
  \label{fig:arch}
\end{figure}
💡
浮动体原理：LaTeX 会根据页面美观度自动决定图/表的最佳位置。如果图片总跑到文档末尾，尝试：
① 使用 [!htbp] 加强位置约束
② 使用 placeins 宏包的 \FloatBarrier 命令阻止浮动体跨越该位置
③ 使用 float 宏包的 [H] 选项强制"就在此处"
🎯 子图（subcaption / subfig） 进阶

💡 一句话：在一个 figure 中并排多个子图，每个子图可独立标注。

\usepackage{subcaption}

\begin{figure}[htbp]
  \centering
  \begin{subfigure}{0.45\textwidth}
    \centering
    \includegraphics[width=\textwidth]{fig1.pdf}
    \caption{子图 A}
    \label{fig:sub_a}
  \end{subfigure}
  \hfill
  \begin{subfigure}{0.45\textwidth}
    \centering
    \includegraphics[width=\textwidth]{fig2.pdf}
    \caption{子图 B}
    \label{fig:sub_b}
  \end{subfigure}
  \caption{总标题}
  \label{fig:main}
\end{figure}

7. 🔗 交叉引用与目录

🎯 label / ref / pageref 基础
📐 语法:
\label{前缀:名称}
\ref{标签} / \pageref{标签}

💡 一句话：LaTeX 的自动引用系统 — 给章节/公式/图/表打标签，编译后自动生成编号，永远不会对错号。

% 打标签（紧跟在编号命令后面）
\section{方法}\label{sec:method}
\begin{equation}
  E = mc^2 \label{eq:einstein}
\end{equation}
\begin{figure}[htbp]
  \caption{架构图}\label{fig:arch}
\end{figure}

% 引用
如第~\ref{sec:method} 节所述...
由公式~\eqref{eq:einstein} 可知...   % eqref 自动加括号
见图~\ref{fig:arch}（第~\pageref{fig:arch} 页）

推荐标签命名约定

前缀对象示例
sec:章节\label{sec:intro}
eq:公式\label{eq:maxwell}
fig:图片\label{fig:result}
tab:表格\label{tab:data}
alg:算法\label{alg:sort}
🎯 目录生成 基础

💡 一句话：LaTeX 自动从章节命令提取标题生成目录，与交叉引用一样需要编译两次。

% 在正文中需要的位置放置
\tableofcontents          % 生成目录
\listoffigures            % 生成插图目录
\listoftables             % 生成表格目录

\setcounter{tocdepth}{3}  % 控制目录深度（3=到 subsubsection）
🎯 hyperref 宏包 基础

💡 一句话：让 PDF 中的交叉引用、目录、URL 变成可点击的超链接。几乎每个文档都该加载。

\usepackage[colorlinks=true, linkcolor=blue, citecolor=green, urlcolor=cyan]{hyperref}

% 🎬 常用选项
% colorlinks=true     彩色链接文字（而不是方框）
% linkcolor=blue      内部引用颜色
% citecolor=green     文献引用颜色
% urlcolor=cyan       URL 颜色
% hidelinks           隐藏所有链接颜色（适合打印）
% pdfstartview=FitH   打开 PDF 时的视图
% bookmarks=true      生成 PDF 书签
⚠️
加载顺序：hyperref 通常是最后一个加载的宏包（少数例外如 cleveref 要在其后）。
🎯 cleveref 宏包（智能引用） 进阶

💡 一句话：自动在引用编号前添加类型名称（如"第 3 节"、"图 5"），无需手动写。

\usepackage{cleveref}   % 必须在 hyperref 之后加载

\cref{sec:method}        % → "section 3" 或 "第 3 节"
\cref{fig:arch,tab:data} % → "fig. 5 and table 2"

% 中文支持
\crefname{section}{节}{节}
\crefname{figure}{图}{图}
\crefname{table}{表}{表}
\crefname{equation}{公式}{公式}

8. 📚 参考文献与引用

🎯 BibTeX 方式 基础

💡 一句话：将文献信息存在 .bib 文件中，LaTeX 自动按指定格式排版引用列表。这是学术界最推荐的方式。

① .bib 文件示例（references.bib）

@article{einstein1905,
  title   = {Zur Elektrodynamik bewegter K{\"o}rper},
  author  = {Einstein, Albert},
  journal = {Annalen der Physik},
  volume  = {322},
  number  = {10},
  pages   = {891--921},
  year    = {1905}
}

@book{knuth1986,
  title     = {The {\TeX}book},
  author    = {Knuth, Donald E.},
  year      = {1986},
  publisher = {Addison-Wesley}
}

② 在正文中使用

% 导言区
\bibliographystyle{plain}   % 引用样式

% 正文末尾（参考文献出现的位置）
\bibliography{references}   % 引用 references.bib（不加 .bib 后缀）

% 正文中引用
爱因斯坦提出了狭义相对论~\cite{einstein1905}。

编译流程

xelatex main        # 第1次编译 → 生成 .aux（含引用请求）
bibtex main         # 处理引用 → 生成 .bbl（含格式化引用列表）
xelatex main        # 第2次 → 将引用列表嵌入文档
xelatex main        # 第3次 → 确保交叉引用编号正确

常用文献类型

类型说明
@article期刊论文
@book书籍
@inproceedings会议论文
@phdthesis博士论文
@misc其他（网页、手册等）
@techreport技术报告
🎯 BibLaTeX（现代替代） 进阶

💡 一句话：BibLaTeX 比传统 BibTeX 更灵活强大，支持更多引用格式，定制更简单。

% 导言区
\usepackage[backend=biber, style=gb7714-2015]{biblatex}  % GB/T 7714 国标
\addbibresource{references.bib}   % 声明文献文件

% 正文中

% 打印参考文献（在需要的位置）

% 编译流程：xelatex → biber → xelatex → xelatex
% 注意后端用 biber 而不是 bibtex

9. ⚙️ 宏与自定义命令

🎯 newcommand 进阶
📐 语法:
\newcommand{\命令名}[参数个数]{定义}
\newcommand{\命令名}[参数个数][默认值]{定义}

💡 一句话：自定义命令，让你用简短命令代替复杂的重复输入。

% 无参数
\newcommand{\R}{\mathbb{R}}       % \R → ℝ（实数集）

% 有参数（#1 代表第一个参数）
\newcommand{\pfrac}[2]{\frac{\partial #1}{\partial #2}}

% 带默认值的可选参数（#1=可选, #2=必选）
\newcommand{\vect}[2][\mathbf]{{#1{#2}}}
% \vect{x}    → \mathbf{x}   （使用默认）
% \vect[\bm]{x} → \bm{x}     （覆盖默认）

% 使用
\pfrac{f}{x}   % 输出 ∂f/∂x
🎯 newenvironment / renewcommand 进阶
📐 语法:
\newenvironment{环境名}[参数个数]{开始定义}{结束定义}
\renewcommand{\已有命令}[参数个数]{新定义}

💡 一句话：定义新环境和重定义已有命令。

% 自定义环境
\newenvironment{proof}[1][证明]   % 默认标题为"证明"
  {\noindent\textbf{#1}:\quad}    % 开始：加粗标题
  {\hfill $\square$\par}          % 结束：证毕符号

% 重定义已有命令
\renewcommand{\abstractname}{摘要}    % 修改摘要标题
\renewcommand{\figurename}{图}        % 修改图标题前缀
\renewcommand{\tablename}{表}         % 修改表标题前缀
🎯 计数器操作 进阶

💡 一句话：LaTeX 用计数器管理章节、公式、图表等的编号。

\newcounter{myctr}                    % 创建新计数器
\setcounter{myctr}{10}                 % 设值为 10
\stepcounter{myctr}                    % +1
\addtocounter{myctr}{5}               % +5
\themyctr                              % 打印计数器值
\arabic{myctr}                         % 阿拉伯数字输出
\roman{myctr}                          % 小写罗马数字输出
\Roman{myctr}                          % 大写罗马数字输出

% 章节编号深度控制
\setcounter{secnumdepth}{3}            % 编号到 subsubsection
\setcounter{tocdepth}{2}               % 目录收录到 subsection

B. 🔧 常见错误与解决

错误信息原因解决方法
Undefined control sequence命令拼写错误或缺少宏包检查拼写；确认已加载相应宏包
Missing $ inserted在文本模式中使用了数学命令把数学符号放到 $...$ 中
File 'xxx.sty' not found缺少宏包用包管理器安装（如 tlmgr install xxx）
There were undefined references标签未定义或引用名写错检查 \label 和 \ref 名称一致；再编译一次
Overfull/Underfull hbox行内容太宽/太窄，排版溢出调整措辞，或允许更多断词（microtype 宏包有帮助）
中文不显示/乱码引擎不支持 UTF-8使用 XeLaTeX 编译 + ctex 宏包
图片不显示路径错误或格式不支持检查路径和扩展名；pdfLaTeX 不支持 EPS
引用显示 [?]BibTeX 未编译或引用键不匹配运行完整编译链；检查 .bib 中的引用键
💡
编译技巧：遇到 "undefined references" 或目录不更新，再编译一次可以解决大部分问题。用 latexmk -xelatex main.tex 可自动处理编译次数。
