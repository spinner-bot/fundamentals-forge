1. 📖 什么是 Markdown

📜 起源与设计哲学

Markdown 由 John Gruber 和 Aaron Swartz 于 2004 年创建。核心理念：

用纯文本标记语法，写出可以轻松转换为 HTML 的文档。即使不渲染，纯文本本身也易读易写。
💡
一句话：Markdown = 用最简单的符号（# * - [ ]）来给文本"排版"，让任何人都能快速写出结构清晰的文档。

三大核心设计原则

原则含义
📖 可读性优先源码本身就应该像纯文本一样易读，而非一堆标签
🔧 易于书写不需要记忆复杂的语法，最常用的格式化只需几个符号
🔄 可转换性可以转换为 HTML、PDF、Word、LaTeX 等多种格式

文件后缀

后缀说明
.md最常用，推荐
.markdown完整拼写，也常见
.mdxMDX 格式（Markdown + JSX 组件），用于 React 文档站点

🌐 Markdown 的"方言"们

原始的 Markdown 功能有限，后来出现了多种扩展版本：

方言全称特色
GFMGitHub Flavored Markdown表格、任务列表、删除线、自动链接、告示块 — 最流行的版本
CommonMark—严格规范，消除歧义。GFM 基于它
MDXMarkdown + JSX可在 Markdown 中使用 React/Vue 组件
R Markdown—嵌入 R 代码，数据科学领域常用
💡
本指南以 GFM（GitHub Flavored Markdown） 为基准，这是目前使用最广泛、支持最丰富的版本。

语法效果说明
**文字**粗体推荐用 **，不用 __
*文字*斜体推荐用 *，不用 _

下标 / 上标 / 键盘键 扩展
📝 Markdown 源码

X^2^（上标）
<kbd>Ctrl</kbd> + <kbd>C</kbd>

⚠️
下标 ~ 和上标 ^ 是部分编辑器（如 Typora）的扩展语法，不是 GFM 标准。GitHub 上不支持，需要用 HTML 标签代替。

7. 📊 表格

基本表格 + 对齐 基础
📝 Markdown 源码
| 左对齐 | 居中对齐 | 右对齐 |
|:-------|:--------:|-------:|
| 内容A  |   内容B  |   100  |

👁️ 渲染效果
左对齐居中对齐右对齐

分隔行写法对齐方式
|:------|左对齐（默认）
|:-----:|居中对齐
|------:|右对齐
💡
表格的列数可以不一致，少的列会自动补空。管道符 | 对齐不是必须的，但强烈建议对齐以提高源码可读性。
