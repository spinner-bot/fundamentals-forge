1. 📖 什么是 Markdown

📜 起源与设计哲学

Markdown 由 John Gruber 和 Aaron Swartz 于 2004 年创建。核心理念：

用纯文本标记语法，写出可以轻松转换为 HTML 的文档。即使不渲染，纯文本本身也易读易写。
💡
一句话：Markdown = 用最简单的符号（# * - [ ]）来给文本"排版"，让任何人都能快速写出结构清晰的文档。

三大核心设计原则

原则含义
📖 可读性优先源码本身就应该像纯文本一样易读，而非一堆标签
🔧 易于书写不需要记忆复杂的语法，最常用的格式化只需几个符号
🔄 可转换性可以转换为 HTML、PDF、Word、LaTeX 等多种格式

文件后缀

后缀说明
.md最常用，推荐
.markdown完整拼写，也常见
.mdxMDX 格式（Markdown + JSX 组件），用于 React 文档站点

🌐 Markdown 的"方言"们

原始的 Markdown 功能有限，后来出现了多种扩展版本：

方言全称特色
GFMGitHub Flavored Markdown表格、任务列表、删除线、自动链接、告示块 — 最流行的版本
CommonMark—严格规范，消除歧义。GFM 基于它
MDXMarkdown + JSX可在 Markdown 中使用 React/Vue 组件
R Markdown—嵌入 R 代码，数据科学领域常用
💡
本指南以 GFM（GitHub Flavored Markdown） 为基准，这是目前使用最广泛、支持最丰富的版本。

2. # 标题与段落

# 标题 Heading 基础

Markdown 支持六级标题，对应 HTML 的 <h1> ~ <h6>。

语法对比

📝 Markdown 源码
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
👁️ 渲染效果

一级标题

二级标题

三级标题

四级标题

五级标题
六级标题
💡
推荐：# 和标题文字之间加一个空格（# 标题），部分解析器对 #标题 不兼容。
习惯：大多数写作者只用前三级标题，深层结构用列表代替。
段落与换行 基础
📝 Markdown 源码
这是第一段。

这是第二段（中间空一行）。

这是同一段内的
直接换行（软换行）。

这是强制换行（行尾两个空格）··
新一行从此开始。
👁️ 渲染效果

这是第一段。

这是第二段（中间空一行）。

这是同一段内的直接换行（软换行）。

这是强制换行（行尾两个空格）
新一行从此开始。

方式写法说明
新段落段落之间空一行产生 <p> 标签，间距较大
软换行直接换行（不空行）同一段落，渲染为空格
强制换行行尾加两个空格再回车产生 <br>，另起一行
GFM 换行直接换行即可在 GitHub 上，段落内直接换行就渲染为 <br>

3. ✨ 文本格式化

粗体 / 斜体 / 删除线 基础
📝 Markdown 源码
这是 **粗体文字**
这是 __也是粗体__

这是 *斜体文字*
这是 _也是斜体_

这是 ***粗斜体***

这是 ~~删除线文字~~

这是 `行内代码`

这是 ==高亮文字==
（部分编辑器支持）
👁️ 渲染效果

这是 粗体文字
这是 也是粗体

这是 斜体文字
这是 也是斜体

这是 粗斜体

这是 删除线文字

这是 行内代码

这是 高亮文字

语法效果说明
**文字**粗体推荐用 **，不用 __
*文字*斜体推荐用 *，不用 _
***文字***粗斜体三个星号
~~文字~~删除线GFM 扩展
`文字`行内代码反引号包裹
==文字==高亮部分编辑器支持（Typora/Obsidian），非标准 GFM
下标 / 上标 / 键盘键 扩展
📝 Markdown 源码
H~2~O（下标）
X^2^（上标）
<kbd>Ctrl</kbd> + <kbd>C</kbd>
👁️ 渲染效果

H2O（下标）

X2（上标）

Ctrl + C

⚠️
下标 ~ 和上标 ^ 是部分编辑器（如 Typora）的扩展语法，不是 GFM 标准。GitHub 上不支持，需要用 HTML 标签代替。
